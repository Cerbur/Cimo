# Plan Step N Template

> 新建 `plan_stepN.md` 时优先复制本模板。本文件只提供可填写的计划骨架；流程规则、编码门禁和归档规则以 [agent_process.md](agent_process.md) 为准。

---

## Step N：阶段名称

### 目标

说明本阶段要解决的真实问题和最终可观察结果。

### 非目标

列出本阶段明确不做的内容，防止实现时顺手扩大范围。

### 当前状态

- Status: Draft / Ready / In Progress / Blocked / Done
- 当前负责人：用户 / 代理 / 待定
- 最近更新时间：YYYY-MM-DD HH:mm 时区

### 背景与约束

- 第一性原理：这个阶段新增的抽象、文件或流程分别解决什么真实问题？
- 奥卡姆约束：哪些更重的方案暂不采用？原因是什么？
- 兼容性约束：哪些旧行为必须保持？
- 依赖约束：依赖哪些已有计划、代码或决策？

### 允许修改范围

- `path/to/allowed/area`

### 禁止修改范围

- `path/to/forbidden/area`

## 验收方式

- 必跑命令：
  - `./gradlew test`
- 场景验证：
  - 描述需要人工或 CLI 验证的关键路径。
- 不通过处理：
  - 记录失败命令、失败现象和下一步处理。
